# 卖家经营玩法、资源不对称与国家平台拓扑

## 核心原则

同一个产品不存在脱离卖家能力的唯一“可做/不可做”。CIDM 先评价产品的客观投资质量，再评价特定卖家的可行域。卖家的供应链、品牌、流量、账号、组织、资本和国家平台网络可以解除部分约束、改变切入方式或降低单位成本，但不能消除真实需求、违法、安全、侵权、平台禁止、不可承受现金流和不可逆声誉红线。

不要再用单一“效率型/精品型/品牌型/规模型”标签完成判断。每次正式报告建立五层画像：

1. **基础经营姿态**：无货源 / 铺货 / 精铺 / 精品 / 品牌 / 规模组合。
2. **不对称能力**：供应链、产品工程、品牌、内容、广告、平台运营、履约、客户、资本、数据技术。
3. **流量与竞争玩法**：搜索承接、推荐流、内容、达人、活动、价格、竞品流量承接、授权跟卖、多链接测试等。
4. **账号与组织拓扑**：单店、多店、品牌矩阵、渠道账号、国家团队和共享中台。
5. **市场拓扑**：单国单平台、单国多平台、多国单平台、多国多平台。

每层记录 `confirmed / partial / assumed / absent`、容量、证据、可解除约束、不可解除约束、成本、依赖和失效条件。

## 约束解除合同

卖家优势只有形成以下闭环，才允许改变“特定卖家可行性”：

`原约束 → 专属资源 → 作用机制 → 容量上限 → 增量成本 → 直接证据 → 新边界 → 失败回滚`

例如“别人做不了，我有供应链”必须继续回答：独家模具还是公开货源、产能多少、交期和良率如何、成本优势是否含模具/质检/账期、竞品多久能复制、放量后是否仍成立。没有证据时只能标记为能力假设。

优势分三类：

- **解除型**：能够真实关闭原约束，例如独家合规产能解决供给稀缺；需要直接证据。
- **缓解型**：降低失败概率或成本，例如成熟广告团队改善学习效率；不能把负贡献变成已盈利。
- **迁移型**：从其他产品、国家或平台迁移能力；默认存在迁移折损，必须用目标对象验证。

产品基础分保持不变。额外输出 `Seller-specific Feasibility`：`Fit / Conditional Fit / Venture Test / No Fit`，并逐项说明是哪些能力改变了可行域。

## 经营玩法矩阵

### 无货源 / 轻资产验证

逻辑：不持有或尽量少持有库存，用供应商直发、代发、预售或本地合作履约换取低失败成本和高测试广度。

- 优势：快速试品、资金占用低、退出快、能覆盖长尾。
- 硬约束：时效、库存同步、包装一致性、售后、平台准入、供应商绕客和质量不可控。
- 可做产品：标准化、低破损、低合规、低退货、供应稳定、时效容忍度高。
- 不能被解除：平台禁止的履约方式、虚假发货、无授权商品、无法控制的安全质量。
- 决策重点：真实履约 SLA、库存接口、订单异常、单笔完全成本和供应商替代，而非只看零库存。

### 铺货 / 数据筛选

逻辑：用高上新吞吐和低单品失败成本获得分布式机会，再把有效 SKU 迁入更深运营。

- 优势：候选覆盖、快速学习、对单品预测依赖低。
- 风险：同质化、运营稀释、账号质量、重复内容和组合现金泄漏。
- 决策重点：组合级最低成功率、单测品成本、清退自动化、SKU 负荷和从铺货到精铺的升级门槛。

### 精铺 / 漏斗式升级

逻辑：宽于精品、深于铺货，先用标准化筛选和轻量测试建立候选漏斗，再把少数通过者升级为中深度运营。

- 优势：兼顾覆盖率和单品深度，适合“多测少留、分层投入”。
- 必要机制：候选池 → 轻测 → 复测 → 精运营 → 清退；每层有资本、时间、数据和库存上限。
- 风险：测试层级不清导致所有 SKU 半深不浅；成功品无法及时获得资源倾斜。
- 决策重点：升级率、单层单位学习成本、资源再分配速度、成功品接管团队和失败品退出时限。

### 精品 / 单品攻坚

逻辑：集中资源完成产品、供应、页面、内容、广告和履约的 0→1，详见 `boutique-zero-to-one-operating-system.md`。

- 优势：能修复可修复缺口，用执行精度提高胜率并以规模换成本。
- 风险：沉没成本、过度自信、单品集中和无限延长修复窗口。
- 决策重点：产品质量卡与组织攻坚卡分离、G0-G10、规模—成本桥和止损权。

### 品牌商家 / 资产复利

逻辑：不是只给商品加 Logo，而是用定位、产品线、证明体系、品牌搜索、渠道一致性、客户资产和价格纪律获得跨周期复利。

- 路线：品类任务与品牌权利 → 英雄品 → 证明资产 → 价格与渠道治理 → 复购/订阅/配件 → 产品线与国家扩张。
- 可接受：前期品牌建设投入和较长回收，但必须区分品牌投资、商品贡献和客户长期价值三本账。
- 可解除约束：信任、溢价、冷启动承接、渠道议价和复购；需要品牌搜索、直接流量、价格保持、客户增量或渠道证据。
- 不能被解除：产品无任务、虚假宣称、不可控质量和长期单位经济无路径。
- 决策重点：英雄品角色、品牌增量而非平台归因、渠道冲突、MAP/价格纪律、品牌资产迁移和失败后资产可回收性。

### 供应链型 / 反向定义产品

逻辑：先从独家材料、工艺、模具、产能、良率、成本或交期优势出发，寻找只有该卖家经济上能成立的市场机会。

- 必须证明：排他性、复制周期、产能、良率、成本全口径、认证、质量和替代技术。
- 可解除约束：MOQ、目标成本、交期、质量稳定、定制深度和规模供给。
- 不能自动解除：需求、内容表达、品牌信任、平台流量和目标市场适配。
- 决策重点：从“市场分数”增加一张供给优势反事实——普通卖家成本下是否 No-Go、本卖家真实成本下是否 Fit；基础市场证据不得删除。

### 规模大卖 / 系统与资本组合

逻辑：用多团队、多账户/渠道、多供应商、共享中台和资本组合提高吞吐、谈判与风险承受，追求组合利润而非每个 SKU 独立最优。

- 优势：跨产品复用、库存调拨、流量数据、采购与物流议价、组织容错。
- 风险：内部竞价、库存和固定成本传染、账号关联、组织复杂度与局部亏损被组合掩盖。
- 决策重点：组合边际、相关风险、共享产能、内部蚕食、国家平台拓扑和资金释放顺序。

## 竞争流量玩法的可行域

### 竞品流量承接（俗称“截流”）

合法合规地承接竞品流量可以包括通用类目词、合法比较型内容、平台允许的商品投放位、差异化替代需求和竞品缺货/涨价窗口。

必须评估：流量来源是否合法、商标使用边界、比较证据、落地页一致性、目标用户为什么切换、转化与边际利润。不得提供商标滥用、虚假比较、误导页面、刷点击、攻击竞品或规避审核方案。

截流优势通常只能缓解获客和冷启动约束，不能证明产品需求或长期留存；竞品恢复、出价上涨或投诉会使窗口失效。

### 跟卖 / 同商品竞争

先区分：

- **授权或合法同款销售**：有真实货权、授权、发票、商品一致性和售后能力；可评估 Buy Box、价格、履约、库存和利润。
- **自有品牌内的渠道协同**：多个授权销售主体按价格和库存规则经营；重点防内耗。
- **侵权、假货或商品不一致跟卖**：直接红线，不提供执行方案。

合法跟卖的可行性来自采购价、履约、库存和账号健康，不来自“抢到链接”本身。必须压力测试价格战、Buy Box 丢失、退货、投诉和授权撤回。

### 多链接 A/B 测试

只有平台允许、商品身份真实一致、测试不会操纵评价/排名/库存且用户体验不受误导时，才能把多链接用于标题、主图、价格、组合或定位实验。优先使用平台原生实验工具；避免把自然波动写成因果。

禁止：重复铺设相同商品操纵搜索、拆分或合并评价、变体滥用、僵尸链接复活、错误类目、虚假库存、规避审核或处罚。正式报告必须列出平台规则核验、实验单位、流量污染、链接资产合并/关闭方案和失败回滚。

### 多店 / AB 店与账号矩阵

多店可能用于品牌/渠道分工、国家实体、本地团队、品类隔离、授权经销或风险隔离，但不能以规避处罚、隐藏关联、重复铺货操纵流量、套利补贴或绕过准入为目的。

评估字段：实际控制人、法律实体、平台授权、品牌/货权、税务、库存所有权、价格政策、数据隔离、客服履约、账号关联和关闭迁移方案。无法证明合规经营目的时标记 `Blocked`，不提供规避关联和风控步骤。

## 国家与平台拓扑

### 单一国家、多平台布局

目标是在同一消费者与供应基础上获取渠道增量、分散平台风险并复用内容/库存/客服。必须检查：

- 用户是否增量还是跨平台比价迁移；
- 平台价格、促销、库存和履约是否冲突；
- 内容、搜索、达人和独立站分别承担什么角色；
- 统一库存池是否导致超卖，退货和客服能否汇总；
- 渠道贡献利润和客户价值是否分别核算。

输出渠道角色图：获客、转化、信任、复购、清仓或测试。不是每个平台都承担完整漏斗。

### 多国家、单平台协同

目标是复用平台系统、商品资产和运营方法，同时通过国家分散生命周期与季节。必须把可复用项和重做项分开：认证、税务、语言、价格、物流、退货、客服、关键词、内容和竞争证据不能直接复制。

使用“母国验证 → 相邻国迁移 → 本地化复验 → 国家组合”的阶段路径；库存、评价、广告和自然排名能否迁移以平台当前规则为准。

### 多国家、多平台协同

先选择母市场、验证市场、规模市场和防御/清仓市场角色，再建立共享中台与本地责任。禁止同时全面铺开。

组合决策至少计算：国家×平台贡献利润、库存可调拨性、合规固定成本、汇率税务、团队容量、渠道蚕食、风险相关性和退出成本。共享资源节省与额外复杂度成本分开列示。

## 输出协议

正式卖家适配报告必须追加：五层画像、玩法主路径与备选路径、能力证据表、约束解除合同、Seller-specific Feasibility、国家平台拓扑、组合协同/蚕食、合法性与平台政策状态、阶段资本/库存上限、成功条件、停止条件和回滚。

对高风险或灰色玩法使用三档输出：`Allowed`（规则明确允许）、`Conditional`（需平台/律师/品牌授权复核）、`Blocked`（侵权、操纵、规避或证据不足）。`Blocked` 只说明风险与合法替代路径，不输出可执行规避步骤。
