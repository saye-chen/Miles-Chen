# 任意国家本地化路由

## 使用场景

当用户指定任何国家，或要求“欧洲、东南亚、中东、拉美、非洲”等区域进入评估时，使用本文件。现有国家 playbook 没覆盖时，本文件是兜底。

正式结论必须落到具体国家、销售平台、进口主体、履约模式、币种和含税/未税口径。区域不是合规或税务结论。

## 十项国家检查

| 模块 | 必查问题 | 证据优先级 |
|---|---|---|
| 消费需求 | 本地使用场景、审美、尺寸、气候、文化、支付意愿 | 本地平台、评论、搜索、社媒 |
| 平台渠道 | 主流平台、账号准入、站内流量、活动机制、线下替代 | 平台官方、卖家中心、本地竞品 |
| 价格税务 | 含税展示、VAT/GST/销售税、平台代征、汇率压力 | 税务机关、平台规则、财税顾问 |
| 产品法规 | 监管分类、认证、标签、说明书、营销宣称、召回责任 | 监管机构、检测机构、平台政策 |
| 进口海关 | HS 编码、关税、进口主体、原产地、许可证、禁限运 | 海关、报关行、官方税则 |
| 物流履约 | 头程、仓配、偏远区、COD、逆向物流、破损和时效 | 物流商、平台仓配规则 |
| 语言内容 | 法定语言、计量单位、客服语言、关键词、本地达人表达 | 平台页面、本地内容、语言规范 |
| 支付售后 | 支付方式、退款习惯、退货地址、客服时效 | 平台规则、本地服务商 |
| 经营能力 | 本地服务商、责任主体、库存、现金周期、退出成本 | 内部资源和服务商报价 |
| 风险边界 | 禁售、侵权、安全、政治/宗教/文化敏感、广告限制 | 官方规则、律师/合规顾问 |

## 区域拆分规则

- 欧洲：至少区分欧盟成员国、英国、瑞士/挪威等非欧盟市场。
- 东南亚：至少区分新加坡、马来西亚、泰国、菲律宾、越南、印度尼西亚。
- 中东：至少区分 UAE、Saudi Arabia、Israel、Turkey 等监管和宗教差异。
- 拉美：至少区分 Mexico、Brazil、Chile、Colombia 等税务和物流差异。
- 北美：美国、加拿大、墨西哥分别处理，不能互套规则。

## 本地化准备度

本地化准备度不计入七维总分，但影响进入顺序。准备度定义与判断标准见 `country-localization.md` 的"国家比较方法"章节（已就绪 / 有条件 / 未就绪三档）。本文件不重复定义，避免两处维护不一致。

## 国家报告输出

1. 国家与销售模式边界。
2. 本地消费者与需求。
3. 渠道结构和平台适配。
4. 法规、税务、进口和标签清单。
5. 本地单位经济和汇率压力场景。
6. 本地化改造：规格、包装、语言、客服、内容。
7. 五道门槛、七维评分、本地化准备度。
8. 进入顺序、暂缓条件和退出路径。

高风险法律、税务、进口和标签问题必须建议当地专业人士复核。
