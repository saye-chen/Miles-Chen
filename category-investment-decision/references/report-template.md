# 报告模板

## 目录

- 通用页眉
- Investment Memo 与压缩卡
- VOC、关键词、国家与平台模板
- 上市后与退出复盘
- 本地 Markdown 交付

先读取 `report-style-guide.md`。本文件定义不同报告的标准版式；具体内容必须结合证据、评分和场景，不机械填空。

默认使用完整专业报告骨架。不同场景不是降低报告完整性，而是在完整骨架中加厚对应模块；快速、卡片、清单和月报也必须保留专业判断闭环。

## 通用页眉

每份正式报告开头必须包含：

| 字段 | 内容 |
|---|---|
| 报告类型 | Investment Memo / Compressed Investment Memo / Operating Review 等 |
| 评估对象 | 品类、商品、链接、店铺、国家、平台或实验 |
| 国家 / 平台 | 必须具体；区域请求需拆成候选国家 |
| 风险偏好 | 合规稳健 / 可控擦边 / 高风险短跑 |
| 模型版本 | `CIDM-2026.14` |
| 证据截止日 | 当前取证日期 |
| 数据口径 | 观察数据、计算结果、分析假设分别说明 |
| 币种和税费 | 统一币种，说明含税/未税 |

## 1. Investment Memo

适用：品类进入、完整投决、投委会、深度研究。

1. 决策页：结论、分数、置信度、投资姿态、红线、下一步动作。
2. 评估边界：产品、国家、平台、生命周期阶段、价格带、卖家画像与已确认资源、风险偏好。
3. 五道门槛表：结论、证据、对决策影响、下一步。
4. 七维评分表：权重、原始分、加权分、置信度、评分依据。
5. STEP1-STEP8：只写关键判断，避免堆砌背景资料。
6. 利润与现金：三场景、敏感性、盈亏平衡 ACOS/CPA、现金周期。
7. 竞品/VOC/关键词：只保留影响切入楔子和评分的证据。
8. 国家/平台适配：当地法规、税务、履约和平台打法。
9. 运营方案：先写生命周期动作上限，再按卖家画像给出人群、SKU、价格、内容、流量、供应链、预算/里程碑；画像为默认假设时不得给出具体备货量或重仓动作。
10. 风险与人工复核：触发条件、角色、材料、进入前条件。
11. Go/Stop：进入条件、放量条件、停止条件。
12. Evidence / Assumption Ledger。

## 2. Compressed Investment Memo

适用：快速判断、初筛、粗看。

1. 一句话结论：四档决策、总分、整体置信度。
2. 用户需求翻译：把原始问题转成商业决策。
3. 门槛速判：五道门槛，每项一句证据和影响。
4. 粗评分与置信度：七维可合并展示，但必须说明评分依据。
5. 利润/风险假设：售价、成本、履约、广告、退货和红线变量。
6. 最大机会与最大风险：切入楔子、反例、关键未知。
7. Go/Stop 与下一步验证：3 个动作，带指标和期限。
8. Evidence / Assumption Ledger：只保留核心判断链。

## 3. Competitor & VOC Intelligence Report

适用：竞品、评论、差评、QA、客服、市场空白。

1. 决策问题和证据范围。
2. 样本说明：原始/去重样本量、来源、日期、偏差。
3. 竞品分层：直接、细分、替代、预算竞品。
4. VOC 六维痛点：主题、严重度、样本内频次、原话摘要、反例。
5. 承诺-体验落差：竞品页面承诺 vs 用户体验。
6. 候选市场空白：特定人群、场景、任务、失败原因。
7. 双轨动作：产品/供应链改进 + Listing/客服/内容优化。
8. 对五道门槛和七维评分的影响。
9. 最弱假设和最小验证。
10. Evidence / Assumption Ledger。

## 4. Keyword Strategy Brief

适用：关键词策略、广告词包、Listing 埋词、流量缺口。

1. 输入范围：ASIN/品类/站点/数据来源。
2. 词簇总览：核心词、长尾词、场景词、噪音词。
3. 八维关键词分类：代表词、意图、商业含义、动作。
4. 机会词清单：证据、竞品覆盖、建议打法、验证指标。
5. 否定词清单：否定原因和适用范围。
6. Listing/广告/内容三端应用。
7. 对评分和门槛的影响。
8. Evidence / Assumption Ledger。

## 5. Country Entry Matrix

适用：国家比较、区域进入、本地化评估。

1. 国家和销售模式边界。
2. 国家矩阵：需求、平台、法规、税务、进口、物流、语言、售后。
3. 本地单位经济：统一币种、汇率、税费、三场景。
4. 本地化准备度：已就绪 / 有条件 / 未就绪。
5. 七维评分和本地化准备度并列展示，不混成一个分数。
6. 进入顺序：先进入、候补、暂缓、停止。
7. 官方来源和人工复核项。
8. Evidence / Assumption Ledger。

## 6. Platform Fit Brief

适用：平台比较、未知平台、平台适配。

1. 平台类型和流量逻辑。
2. 账号/类目准入。
3. 费用、履约和售后。
4. 内容/Listing/广告打法。
5. 竞争结构和新品突破。
6. 平台风险：限流、下架、扣分、资金冻结。
7. 测款指标、放量条件、停止条件。
8. Evidence / Assumption Ledger。

## 7. Post-launch Operating Review

适用：上市后放量、稳态、衰退、老品诊断。

1. 当前阶段：放量期 / 稳态期 / 衰退期。
2. KPI 面板：ACOS/CPA、CVR、退货、星级、库存、排名。
3. 告警等级：绿灯 / 黄灯 / 红灯 / 机会。
4. 根因树：流量、转化、竞品、评论、广告、季节、库存。
5. 决策：`Scale / Maintain / Review / Reduce / Stop`。
6. 动作表：动作、负责人角色、指标、期限、复查日期。
7. 是否触发重新评分。
8. Evidence / Assumption Ledger。

## 8. Exit Review Memo

适用：失败复盘、停止、清退。

1. 初始假设和评分版本。
2. 实际表现：流量、转化、利润、退货、评价、库存、现金。
3. 被验证/部分验证/推翻的假设。
4. 损失核算：广告、库存、折扣、退货、仓储、机会成本。
5. 根因树：需求、定向、素材、价格、页面、产品、履约、合规。
6. 可复用学习。
7. 下次前置规则：红线、样本门槛、停止条件。
8. Evidence / Assumption Ledger。

## 本地 Markdown 交付

用户明确要求保存、报告文件或指定格式时，使用上述模板生成对应交付物，并包含 Evidence / Assumption Ledger。demo、压测或看效果但未要求文件时在对话中交付。

若用户未指定路径，保存到 `$HOME/Downloads`。写入前按主 Skill 同名规则检查。
