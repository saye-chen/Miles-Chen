# 精品型单品 0→1 攻坚与规模降本协议

## 适用边界

当卖家以少量 SKU、充分准备、深度运营和单品突破为主要经营方式，且要求调研后继续形成打样、测品、测款、采购准备、备货、上市、放量、稳态与退出计划时使用本协议。

本协议不把“运营强”写成产品必胜，也不让执行信心突破违法、安全、侵权、平台禁限、不可承受现金流或不可逆库存红线。CIDM 负责资本姿态、阶段资源上限、里程碑和最终 Go/Stop；广告内部预算/出价交给 D09，内容机制交给 VLB，采购、物流、Listing、达人、CRM 等专业域以结构化 Brief 和验收合同承接，不由 CIDM伪造专业结论。

## 双层判断：产品质量与组织胜率分开

精品型报告必须同时输出两张卡，禁止只给一个总分：

1. **产品先天投资质量卡**：继续使用五道门槛、七维评分、证据置信度和利润模型，回答对象本身是否值得投。
2. **精品组织攻坚卡**：回答卖家能否通过产品改造、供应整合、运营精度、内容、投放、评价、履约和规模谈判，把可修复缺口变成经营优势。

组织能力不能直接加到 CIDM 七维分数，也不能抵消硬红线。它只影响：是否值得进入受控攻坚、验证顺序、资源释放速度、允许修复窗口和成功概率区间。

### 精品组织攻坚卡

| 能力域 | 必须取证的资源 | 可形成的优势 | 失效信号 |
|---|---|---|---|
| 产品工程 | 打样工程师、模具/结构/材料能力、测试设备 | 把 VOC 缺口转成可验证规格 | 只能换包装，关键缺陷不可改 |
| 供应链议价 | 供应商池、阶梯报价、产能与账期 | 随量降低采购成本、质量成本 | 降价以偷料或超 MOQ/现金上限为代价 |
| 运营系统 | 专职负责人、日报/周会、关键词与页面能力 | 快速学习和修复承接 | 无负责人、动作无法归因 |
| 内容与广告 | 素材产能、测量能力、预算纪律 | 建立多增长端口 | 只靠单素材、单词或不可复现爆量 |
| 履约与售后 | 仓网、补货、质检、客服与逆向 | 降低交付和失败成本 | 放量后时效、退货和差评失控 |
| 资本与组织 | 可承受损失、现金周期、审批与止损权 | 给学习足够窗口 | 用其他 SKU 输血且无退出时间 |

每个能力域标记 `confirmed / partial / assumed / absent`，附证据、容量、负责人和推翻条件。用户只说“我们运营很强”时只能标记为 `assumed`。

## 低分品的精品攻坚例外

常规 CIDM 结论不因卖家类型改变。只有同时满足以下条件，`50–64.9` 的“仅观察”候选才可增加一个独立的 `Boutique Venture Test` 选项：

- 没有违法、安全、侵权、平台禁止或不可逆声誉红线；
- 主要失分来自可修复项，而非需求不存在、结构性负贡献或产品本质缺陷；
- 至少一个目标对象直接证据支持切入楔子；
- 精品组织攻坚卡至少四个能力域为 `confirmed`，其余不得为关键缺口；
- 已确定可承受损失、现金周期、单阶段资本上限和库存退出路径；
- 所有“规模后会降本”的收益先作为假设，初始决策仍按当前可得成本计算；
- 采用阶段闸门释放资源，不允许因信心一次性备大货。

`<50`、不可补偿红线、压力场景结构性负贡献或需求证伪仍为 Stop。例外必须单列，不得把原 CIDM 结论改写成“建议进入”。

## 调研后交付：0→1 主计划

精品模式的完整调研不得止于“值不值得做”。当用户要求完整报告或确认进入攻坚，追加一份主计划，至少包含以下工作流、依赖、责任人、预算上限、交付物、验收、Go/Stop 和回滚。

| 阶段 | 核心问题 | 必要交付物 | 释放下一阶段的条件 |
|---|---|---|---|
| G0 投资立项 | 为什么是这个产品、谁负责、最多亏多少 | 投资命题、最弱假设、资本与时间上限、退出路径 | 红线清零，资源责任人确认 |
| G1 规格冻结 | VOC 如何变成可生产差异 | PRD、CTQ、目标成本、禁止降级项、竞品对照 | 可测规格、合规路径和目标成本成立 |
| G2 多轮打样 | 样品是否解决任务且可稳定制造 | 样品版本台账、工程/用户/运输/合规测试 | 核心任务、质量和安全门槛通过 |
| G3 测品 | 产品体验、退货风险和证明链是否成立 | 盲测/场景测试、缺陷 FMEA、说明书与包装修订 | 决定性缺陷关闭，证明可复现 |
| G4 测款 | 哪种价格、页面、内容和流量组合成立 | 预注册测试矩阵、落地页/Listing Brief、素材矩阵、经济结果 | 成熟贡献与关键护栏通过 |
| G5 首批采购与备货 | 多少货既能验证又不致命 | 供应商选择合同、质检方案、首批量、补货点、退出渠道 | 现金、交期、良率和退出库存可承受 |
| G6 上市冷启动 | 从 0 获得有效数据和首批口碑 | 上市日历、流量结构、内容节奏、客服与异常 SOP | 数据/追踪成熟，核心指标进入学习轨道 |
| G7 放量攻坚 | 如何打排名、扩流量并守住利润 | 周级作战图、预算/库存阶梯、素材与词包迭代、战情室 | 边际利润、质量、库存和组织容量同时通过 |
| G8 规模降本 | 量能否转化为真实单位成本优势 | 阶梯成本水位、谈判包、物流方案、良率与售后成本曲线 | 节省已落合同/账单且未破质量/现金护栏 |
| G9 稳态/扩张 | 单点胜利是否可持续复制 | 稳态 P&L、补货政策、变体/站点扩张条件、风险监控 | 多窗口成熟且增长端口不过度集中 |
| G10 修复/退出 | 失败后如何保本金和学习 | 修复时限、清仓/转渠道/拆件方案、复盘与资产回收 | 到期必须 Repair、Reduce 或 Stop，不无限续期 |

## 超级精细运营作战图

正式 0→1 计划按“阶段—周—日—事件”四层展开，但不编造平台战术：

- **阶段层**：目标、核心假设、资本上限、库存上限、里程碑和退出条件。
- **周层**：产品、供应、页面、内容、广告、履约、评价、客服、财务九条工作流的目标与依赖。
- **日层**：上线前后关键动作、负责人、输入、输出、审批和数据截止时间。
- **事件层**：断货、转化骤降、差评集中、广告失速、物流异常、竞品降价和平台审核的响应 SOP。

每个动作使用统一字段：`action_id / owner / dependency / start / deadline / budget_cap / deliverable / acceptance / metric / success / stop / rollback / evidence_writeback`。没有负责人、截止日、验收和回滚的动作不进入主计划。

### 九条并行工作流

1. 产品与打样：需求冻结、版本管理、CTQ、FMEA、签样和变更控制。
2. 合规与知识产权：法规、认证、标签、宣称、专利/商标和人工复核。
3. 采购与质量：供应商竞争、阶梯报价、账期、AQL/关键测试、批次追溯。
4. 包装与物流：体积重量、破损、头尾程、仓储、补货和逆向退出。
5. 页面与转化：定位、价格、信息架构、证明资产、FAQ 和本地化；由相应专业域形成执行稿。
6. 内容与传播：Hook、证明、素材组合、疲劳和迁移；由 VLB 提交机制与测试合同。
7. 广告与流量：追踪、结构、预算/出价、边际与放量；由 D09 保有动作主权。
8. 评价、客服与体验：种子反馈合规边界、问题闭环、退货、差评和服务恢复。
9. 财务与战情室：三本账、现金、库存、决策日志、日报、周复盘和升级机制。

## 规模倒逼降本：先建桥，再兑现

精品大卖可以用预期销量换取采购、物流、包装、仓储、广告生产和售后效率，但必须建立“规模—成本桥”，禁止先把未来低成本写进当前利润。

`规模后单位贡献 = 售价 - 平台与税费 - 成熟退款 - 广告/获客 - 采购 - 头程 - 履约 - 仓储 - 质量失败 - 售后 - 资金成本`

对每一项降本记录：当前水位、目标水位、触发销量/采购量、供应商或承运商书面依据、一次性投入、质量/时效代价、生效日期、可逆性和 owner。

### 成本释放顺序

1. 先消除浪费：包装体积、破损、返工、滞销仓储、紧急空运和无效广告。
2. 再做竞争性采购与物流组合，不以单一供应商锁定换表面低价。
3. 再用稳定预测和订单换阶梯价、账期、产能与自动化。
4. 最后评估模具、专线、海外仓或垂直整合等不可逆投入。

只有合同、发票、账单或稳定批次实际数据验证的节省才能进入 `validated cost`；报价和口头承诺保持 `proposed`。任何降本若导致材料降级、良率下降、退货上升、交期变长或现金占用恶化，按净贡献重算，不能只看采购单价。

## 战情室与节奏

- 日会只处理红灯、阻塞和当天决策，不做无关汇报。
- 周会复核假设、证据成熟、三本账、库存覆盖、九条工作流和下周资源释放。
- 每个 Gate 召开正式复核：继续、迭代、降级、暂停或退出；不允许“再看看”而没有新证据与截止日。
- 重大版本、供应商、价格、页面、内容或流量结构同期变化时，记录混杂并降低归因强度。

## 必要输出

精品型完整报告至少追加：产品质量卡、组织攻坚卡、G0-G10 主计划、九工作流 RACI、周级节奏、阶段资本/库存阶梯、规模—成本桥、异常 SOP、成功条件、停止条件、回滚和证据回写表。数据不足时可以把数字保留为待确认变量，但不能删除模块或虚构资源。
