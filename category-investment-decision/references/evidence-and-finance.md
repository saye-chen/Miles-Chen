# 证据与利润口径

## 证据等级

| 等级 | 类型 | 示例 | 使用限制 |
|---|---|---|---|
| A | 官方/一手 | 监管机构、平台政策、品牌或商品页、用户提供的后台数据 | 可支撑关键事实；仍记录日期与适用范围 |
| B | 高质量代理 | 多个真实商品页、Google Trends、供应商报价、公开排名/评论结构 | 可支撑方向判断，避免过度推算精确销量 |
| C | 二手研究 | 可靠行业报告、媒体、研究机构摘要 | 用于市场背景，关键结论需交叉验证 |
| D | 弱信号 | 聚合站、论坛单帖、无来源数字 | 只用于提出假设，不作为进入依据 |

每条关键证据使用：`[观察数据｜等级｜截至日期]`。模型计算使用 `[计算结果]`，未验证输入使用 `[分析假设]`。

引用直接支持对应结论的页面。法律、政策和平台费优先引用官方来源。注明地区、平台、币种和时间范围。

## 单位经济模型

统一以单件、成交含税/不含税口径中的一种计算，并说明口径。基础公式：

`净利 = 售价 - 产品 - 包装 - 关税 - 头程 - 履约/尾程 - 平台佣金 - 广告 - 促销/联盟 - 退货准备金 - 仓储 - 质检损耗 - 支付/汇兑 - 其他`

`净利润率 = 净利 ÷ 售价`

`盈亏平衡广告率 = 扣除广告前贡献利润 ÷ 售价`

注意：销售税/VAT 若由平台代征且不进入卖家收入，不要重复作为成本；若报价或结算包含税费，明确调整。

## 固定成本 / 可变成本分层

测品、小批量、达人视频、直播和备大货阶段必须区分单件可变成本与批次固定成本。不要把样品、达人合作费、视频制作费、固定投流、人工分摊等沉没投入混进单件采购成本；也不要把平台佣金、达人佣金、支付手续费、物流和退货损耗漏出单件贡献利润。

| 成本层级 | 常见项目 | 使用方式 | 决策含义 |
|---|---|---|---|
| 单件可变成本 | 采购、包装、头程均摊、尾程/履约、平台佣金、达人佣金、支付费、退货损耗 | 计算单件贡献利润和盈亏平衡 ACOS/CPA | 决定每新增一单是否赚钱 |
| 批次固定成本 | 样品、达人合作费、视频制作、固定投流、人工/管理分摊、打样和质检 | 计算本轮测品或小批量需要多少订单回本 | 决定本轮投入是否值得释放 |
| 放量固定/准固定成本 | MOQ、库存资金占用、仓储、团队人工、认证、设备、长期内容产能 | 放量前做现金流和库存压力复核 | 决定是否能备大货或长期经营 |

统一公式：

`单件贡献利润 = 售价 × (1 - 平台佣金率 - 达人佣金率 - 支付费率 - 退货损耗率 - 促销/联盟率) - 单件产品/包装/物流/履约/其他可变成本`

`批次盈亏平衡订单量 = 批次固定成本 ÷ 单件贡献利润`

`批次盈亏平衡销售额 = 批次盈亏平衡订单量 × 售价`

当单件贡献利润小于等于 0 时，不应通过增加曝光、达人或库存解决；先修复售价、成本、佣金、物流或退货假设。需要重复计算时运行 `scripts/profit_model.py`，使用 `--batch-fixed-costs` 或 `--batch-fixed-costs-json` 记录批次固定成本。

## 反向漏斗推导

反向漏斗用于把财务目标翻译成运营目标，尤其适用于 TikTok Shop 达人寄样、短视频投流、小批量测试和直播场次复盘。它回答的不是“已有数据是否达标”，而是“投放前至少要做到多少订单、点击和曝光才不亏”。

核心公式：

`最低订单量 = 批次固定成本 ÷ 单件贡献利润`

`最低点击量 = 最低订单量 ÷ CVR`

`最低曝光量 = 最低点击量 ÷ CTR`

若已知预计曝光，也可反推：

`可承受最高固定成本 = 预计曝光 × CTR × CVR × 单件贡献利润`

使用规则：

- CTR、CVR 必须来自同账号、同平台、同类目或显式假设，不使用全局硬编码行业线。
- 达人端口要单独计算单达人固定成本，再汇总多达人组合固定成本。
- 反向漏斗通过不等于可以重仓；仍需验证合规、供应链、退款、内容可复制和样本量。
- 反向漏斗不修正基础评分，只把利润/渠道假设变成可执行门槛和停止条件。

结构化输入时运行 `scripts/reverse_funnel.py standard input.json` 或 `scripts/reverse_funnel.py creator input.json`。

## 阶段化损益分类

从千点白数据测算模型吸收的阶段化口径如下。报告中不需要机械展开所有阶段，但当用户做测款、备货、达人、直播或放量决策时，至少选择对应阶段复核。

| 阶段 | 核心收入/成本口径 | 必看输出 | 常见误判 |
|---|---|---|---|
| 选品测品 | 单测品成本、点击/转化、初步 ROI | 是否值得进入样品或小批量 | 用点击替代购买，用静态阈值替代类目基准 |
| 小批量备货 | Revenue、COGS、Logistics、Marketing、PlatformFee、Labor、ReturnCost、Profit | 售罄率、退货率、净利润和 ROI | 只看 GMV，不看退货、样品、人工和达人分佣 |
| 达人视频 | 单达人固定成本、佣金、内容 CTR/CVR | 单达人端口最低订单/曝光 | 用播放量替代商品点击和成交 |
| 备大货 | MOQ、库存资金、采购合格率、物流时效、售后成本、营销预算 | 月净利润、库存覆盖和现金回收 | 小批量盈利后直接重仓，忽略现金和售后 |
| 直播 | UV、停留、互动、点击、加购、下单、支付、单场成本 | 单场净利润和全链路转化 | 用直播间热度替代利润，用极端折扣制造假转化 |

## 三种场景

- 基准：最可能的售价、广告、退货和履约成本。
- 压力：售价下降、广告率上升、退货/损耗和物流至少两项恶化。
- 乐观：合理但非极端的提价、降本或转化改善。

至少报告基准和压力场景；完整报告同时报告乐观场景。敏感性优先检查广告率、售价、退货率、履约/头程。

## 敏感性分析模板

完整报告必须在三场景之外，对关键变量做单变量敏感性。每次只改变一个变量，其余保持基准场景不变；如果变量天然不是线性关系，说明计算假设。

| 变量 | -10% 净利率 | 基准净利率 | +10% 净利率 | 对结论的影响 | 修复动作 |
|---|---:|---:|---:|---|---|
| 售价 |  |  |  | 是否跌破目标净利或价格带 | 提价、套装、规格调整 |
| 广告率 |  |  |  | 是否接近盈亏平衡广告率 | 内容优化、长尾词、达人/联盟 |
| 退货率 |  |  |  | 是否触发退货或评价红线 | 包装、说明书、质检、预期管理 |
| 采购成本 |  |  |  | 是否过度依赖供应商报价 | 备选供应商、MOQ 梯度、材料替代 |
| 头程/履约 |  |  |  | 是否受尺寸、重量或渠道波动影响 | 包装压缩、渠道切换、本地仓 |

敏感性结论至少回答：

- 哪个变量最容易把利润从“可进入”推到“不可进入”。
- 压力场景亏损是否有可执行修复路径，修复路径依赖什么验证。
- 若任一关键变量 10% 恶化即触发亏损或停止条件，利润空间维度不得高于 6 分，除非有已验证的一手成本或转化数据。

## 人工复核触发规则

选品报告可以做商业初筛，但以下事项不得只靠模型结论放行。

| 触发类型 | 强制触发条件 | 建议复核角色 | 复核输出 |
|---|---|---|---|
| 合规/安全 | 涉及食品接触、儿童、个护美妆、医疗健康、电子电器、化学品、危险品、承压/加热/带电产品 | 目标国家检测机构、合规顾问、平台规则负责人 | 适用法规、认证路径、标签/宣称限制、测试成本与周期 |
| IP/品牌 | 外观、结构、图案、角色、品牌词、专利或版权素材存在相似风险 | 商标/专利代理人、法务、资深平台运营 | 商标/专利/FTO 初筛结果、禁用词、可替代设计 |
| 财税/海关 | 税率、申报、VAT/GST/销售税、关税归类或进口限制会影响毛利或可售性 | 财税顾问、报关行、当地合规顾问 | 税费口径、HS 编码假设、申报风险、成本修正 |
| 利润/供应链 | 总分 80+ 但利润或供应链维度置信度为“不确定”；压力场景亏损但声称可修复 | 供应链负责人、财务负责人、采购负责人 | 报价、样品、包装尺寸、交期、MOQ、质检点和修复成本 |
| 平台政策 | 涉及禁限售、功效宣称、评价/售后高风险或账号能力边界 | 平台运营负责人、合规负责人 | 是否可上架、所需材料、审核失败预案 |

复核前只能给“条件进入 / 小规模测试 / 暂缓评分”结论。复核完成后，更新对应维度置信度、成本假设和红线状态；不要只在风险段落追加一句“已复核”。

## 高风险收益观察池

当用户明确接受擦边、高处罚概率或短周期打法时，增加高风险收益观察池。观察池用于商业判断，不等同于推荐进入，也不得提供规避审核、误导平台或隐藏真实用途的操作建议。

| 字段 | 要求 |
|---|---|
| 高风险机会 | 写清产品、平台、国家和触发风险的具体原因 |
| 潜在收益 | 估算毛利、内容爆发力、竞争稀缺性、短期现金回收 |
| 处罚成本 | 估算下架、扣分、账号权限损失、达人损失、库存滞销、退款和资金冻结 |
| 内容寿命 | 估算素材可持续周期、审核波动和复用难度 |
| 库存退出 | 写清转平台、转渠道、改包装、清仓或报废路径 |
| 替代渠道 | 给出更适合的合规渠道或平台，不把高风险平台强行洗白 |
| 决策姿态 | 只能使用“观察 / 单独测算 / 高风险短跑 / 停止”，不得写成“合规进入” |

高风险收益池与合规推荐分开排序：合规候选按七维模型排序；高风险候选按“预期收益 - 处罚成本 - 库存退出成本”解释，不并入基础总分。

## 阈值使用

搜索量 1 万、净利率 20%、评论 2,000 条等都是警戒线，不是普适真理。必须结合：

- 平台：搜索电商、内容电商或独立站；
- 产品：标品/非标、高低客单、耗材/耐用品、季节性；
- 卖家：资金、供应链、内容能力、品牌与合规能力；
- 目标：快速现金流、规模增长、订阅/LTV 或品牌资产。

若偏离默认阈值，写清补偿逻辑、验证指标和停止条件。
