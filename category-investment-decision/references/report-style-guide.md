# 报告专业交付规范

> **本文件 vs `report-template.md`**：本文件定义报告的专业标准——五层结构、决策页规范、表格规范、Ledger schema、表达规则和文件交付规则。`report-template.md` 定义每种报告类型的具体章节骨架。两者互补：先按本文件确保专业质量，再按 `report-template.md` 选择对应骨架。

## 目录

- 目标、五层结构与高密度原则
- 决策页与表格规范
- Evidence / Assumption Ledger
- 表达、报告类型与文件交付

## 目标

每份报告都必须像可交给老板、投委会或业务负责人阅读的专业交付物，而不是聊天摘要。报告要让读者快速知道：结论是什么、证据够不够、钱能不能赚、风险在哪里、下一步做什么、什么情况下停止。

## 五层结构

所有报告默认按五层组织；快速版可以压缩背景和附表，但不能缺失决策、证据、推理、利润、风险和动作。

| 层级 | 作用 | 必须包含 |
|---|---|---|
| 结论层 | 让读者 30 秒知道怎么决策 | 四档结论、总分、整体置信度、投资姿态、红线、下一步 |
| 证据层 | 说明依据来自哪里 | 关键观察、证据等级、日期、来源、缺口 |
| 推理层 | 把证据连到商业判断 | 五道门槛、七维评分、反例、冲突证据、假设 |
| 行动层 | 转成执行计划 | 验证动作、预算/周期、负责人角色、Go/Stop 条件 |
| 附录层 | 保留审计线索 | Evidence/Assumption Ledger、计算口径、原始样本摘要、复核项 |

## 高密度专业报告

对外交付永远保持专业完整，对内执行使用时间盒和决策级深挖。报告要压缩低价值背景，不压缩关键判断。

| 模块 | 决策级深挖要求 |
|---|---|
| 市场需求 | 覆盖规模、稳定性、购买动机、搜索/内容信号，并说明是否足以支撑目标规模 |
| 竞品 | 先分层头部、腰部、新品/细分突破，再抽样深拆代表竞品和进入缺口 |
| VOC | 输出前三大痛点、严重度、反例、产品化机会和评分影响；专项 VOC 才做完整原子编码 |
| 利润 | 基准、压力、乐观三场景，关键变量敏感性，盈亏平衡 ACOS/CPA 和推翻变量 |
| 合规 | 红线、平台禁限、品类相关规则入口、人工复核项和进入前材料 |
| 行动 | SKU、定价、内容、流量、供应链、测试路径和 Go/Stop 条件 |

深挖范围由“是否可能改变结论”决定。会改变进入、投资额、切入楔子、停止条件或渠道选择的点必须深挖；只增加背景感的资料应压缩。

## 决策页

完整报告、压测报告、投委会报告和本地 Markdown 交付必须以“决策页”开头。

| 字段 | 要求 |
|---|---|
| 最终结论 | 只能使用四档决策之一，或经营场景中的 Scale/Maintain/Review/Reduce/Stop |
| 投资姿态 | 重仓、条件进入、小规模测试、观察、停止、清退等 |
| 总分与置信度 | 写明 `CIDM` 版本、总分、整体置信度、维度置信度短板 |
| 关键红线 | 合规/IP/安全/利润/现金流/供应链中最可能推翻结论的项 |
| 一句话理由 | 不超过两句，说明为什么不是另一个结论 |
| 下一步动作 | 3-5 个动作，带指标、期限或触发条件 |

## 表格规范

报告中的核心表格必须服务决策，不做资料堆砌。

| 表格类型 | 必备列 |
|---|---|
| 门槛表 | 门槛、结论、关键证据、对决策影响、下一步 |
| 评分表 | 维度、权重、原始分、加权分、置信度、评分依据 |
| 风险表 | 风险、触发条件、影响、复核角色、进入前动作 |
| 行动表 | 动作、目标指标、周期、负责人角色、Go/Stop |
| 竞品表 | 竞品、定位、变化/卖点、威胁/机会、我方动作 |
| 国家/平台表 | 国家/平台、机会、成本/风险、准备度、进入顺序 |

表格后必须写 1-3 句“所以怎样影响决策”。不要让读者自己从表格里猜结论。

## Evidence / Assumption Ledger

正式报告的来源区使用 Evidence / Assumption Ledger，而不是只列链接。Evidence 部分收录支持核心判断链的证据；Assumption 部分收录尚未核验但用于计算或判断的关键假设。

| Evidence 字段 | 要求 |
|---|---|
| 证据编号 | `E1/E2/...` |
| 来源 | 标题、平台或机构、链接 |
| 日期 | 访问日期或数据日期 |
| 等级 | A/B/C/D |
| 支持判断 | 支持哪个门槛、评分或风险 |
| 局限性 | 是否代理信号、样本偏差、不可访问字段 |

| Assumption 字段 | 要求 |
|---|---|
| 假设编号 | `A1/A2/...` |
| 假设内容 | 售价、成本、转化、退货、合规路径、供应链等未核验变量 |
| 使用位置 | 哪个计算、评分或决策使用了该假设 |
| 推翻条件 | 哪个数据会改变结论 |
| 补数动作 | 谁在何时用什么材料补齐 |

关键结论至少要能回指一个 Evidence 编号或 Assumption 编号。证据数量不硬卡；每条证据必须服务结论、评分、红线或动作，避免把来源区变成资料库。

## 专业表达规则

- 使用“观察数据 / 计算结果 / 分析假设 / 决策含义”区分信息类型。
- 不写“有潜力、竞争激烈、建议优化、提升质量、做品牌”等空话，除非后面紧跟具体证据和动作。
- 不把高分写成重仓理由；必须同时看置信度、红线和现金占用。
- 不把低置信度写成确定结论；低置信度高分最高只能支持小规模测试。
- 不把行业常识当成当前事实；价格、费率、法规、物流和平台规则需要当前来源。
- 失败、停止和不建议进入也是专业结论，不为了显得积极而软化。
- 不把高风险或不合规需求简单拒绝为无输出；必须转成合规路径、高风险收益观察池、替代渠道、处罚成本、退出路径和人工复核项，但不得提供违规执行步骤。

## 报告类型

| 报告类型 | 使用场景 | 专业重点 |
|---|---|---|
| Investment Memo | 品类进入、完整投决 | 决策页、五道门槛、七维评分、利润、风险、运营打法 |
| Compressed Investment Memo | 快速初筛、粗看、卡片化要求 | 结论、门槛速判、粗评分、利润/风险假设、关键证据、Go/Stop |
| Competitor & VOC Intelligence Report | 竞品/VOC/市场空白 | 样本范围、痛点证据、反例、切入楔子 |
| Keyword Strategy Brief | 关键词/广告/Listing | 词簇、意图、机会词、否定词、三端应用 |
| Country Entry Matrix | 国家比较 | 本地化准备度、法规税务、履约、进入顺序 |
| Platform Fit Brief | 平台比较/平台适配 | 流量逻辑、费用、履约、内容/搜索打法 |
| Post-launch Operating Review | 上市后巡检 | KPI、告警、根因、Scale/Maintain/Review/Reduce/Stop |
| Exit Review Memo | 退出复盘 | 初始假设、实际表现、损失、学习和下次规则 |

## 本地文件交付

只有用户明确要求保存、生成报告文件或指定格式时才生成本地文件。“demo”“压测”或“看一下效果”本身不构成写文件授权；未要求文件时在对话中交付。

文件内容必须包含决策页、正文、Evidence/Assumption Ledger 和自检摘要；不得只把聊天内容原样复制成 Markdown。
