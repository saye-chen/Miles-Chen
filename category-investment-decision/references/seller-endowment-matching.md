## 使用原则

CIDM 七维评分模型保持通用——不因卖家类型改变评分锚点或门槛红线。卖家画像的作用在评分之后的**决策解读、运营方案和资源匹配**层：同一品类、同一分数，对不同卖家原型的投资建议、进入方式、SKU 策略和资源配置方案不同。

卖家画像在三个环节介入：

1. **开始前**：作为结构化输入，影响报告的重点模块和取证优先级。
2. **评分后**：影响置信度解读、投资姿态映射和运营方案差异化。
3. **组合决策**：影响硬淘汰（与卖家能力明显不匹配）、组合约束和资源协同判断。

## 四类卖家原型

四类原型只用于描述基础资源姿态，不再代表完整经营玩法。正式判断必须叠加 [seller-operating-models-and-topologies.md](seller-operating-models-and-topologies.md) 的五层画像：基础经营姿态、不对称能力、流量与竞争玩法、账号与组织拓扑、国家平台拓扑。分类依据是核心驱动力和资源约束，不是绝对 SKU 数或团队规模。

### 效率型（对应铺货/精铺）

| 维度 | 特征 |
|---|---|
| 预算区间 | 1-10 万 |
| 团队规模 | 1-5 人 |
| SKU 容量 | 50-500 |
| 供应链深度 | 现成模具/公模为主，少量微调 |
| 核心能力 | 上架效率、多平台铺货、活动运营、价格竞争 |
| 决策关注点 | 周转速度、上架门槛、活动适配性、价格带竞争力 |
| 风险承受 | 单品投入低但总量大，靠成功率分散风险 |
| 内容能力 | 弱，依赖平台自然流量和活动，或低成本批量素材 |

### 精品型

| 维度 | 特征 |
|---|---|
| 预算区间 | 5-30 万 |
| 团队规模 | 3-15 人 |
| SKU 容量 | 10-100 |
| 供应链深度 | 可开私模、可做材料/工艺/包装定制 |
| 核心能力 | 选品深度、Listing 优化、广告投放、Review 运营 |
| 决策关注点 | 差异化楔子、Review 壁垒可攻破性、广告 ROI、利润深度 |
| 风险承受 | 单品投入中等，靠深度选品和运营精细度取胜 |
| 内容能力 | 中等，能做产品拍摄和基础视频，达人合作有限 |

精品型不等于保守的小卖家。成熟精品大卖通常以少量核心 SKU 集中产品、供应链、运营、内容和资本资源，接受受控失败风险，并试图通过高强度执行与销量规模重构单位成本。涉及此类卖家时必须读取 [boutique-zero-to-one-operating-system.md](boutique-zero-to-one-operating-system.md)，不能只输出“小批量测款”。

### 品牌型

| 维度 | 特征 |
|---|---|
| 预算区间 | 20-100 万+ |
| 团队规模 | 10-50 人 |
| SKU 容量 | 不限，按产品线规划 |
| 供应链深度 | 深度定制、自有或独家供应链、品控体系 |
| 核心能力 | 品牌叙事、内容营销、全渠道运营、用户运营 |
| 决策关注点 | 品牌一致性、LTV/复购、内容传播性、品类延伸空间 |
| 风险承受 | 可承受较长回报周期，靠品牌资产和用户积累取胜 |
| 内容能力 | 强，有内容团队或稳定达人矩阵，能做品牌故事 |

### 规模型（对应大卖）

| 维度 | 特征 |
|---|---|
| 预算区间 | 50 万-数百万+ |
| 团队规模 | 30+ 人 |
| SKU 容量 | 数百-数千 |
| 供应链深度 | 多品类供应链网络、可能自有工厂或深度绑定 |
| 核心能力 | 组合管理、资本效率、多平台/多站点运营、组织体系 |
| 决策关注点 | 组合收益、资金效率、风险分散、组织承载力 |
| 风险承受 | 可承受多品类并行测试，靠系统和规模取胜 |
| 内容能力 | 中到强，有专职内容团队，但更依赖系统化素材生产 |

## 卖家画像输入

开始前确定卖家画像时，收集以下结构化字段。字段不完整时使用显式假设继续，并在报告中注明。

| 字段 | 选项/格式 | 用途 |
|---|---|---|
| 卖家原型 | 效率型 / 精品型 / 品牌型 / 规模型 | 决策解读、运营方案差异化 |
| 可用预算 | 金额 + 币种 | 组合约束、投资姿态映射 |
| 当前 SKU 数 | 数字 | 判断运营负荷承载力 |
| 团队规模 | 人数 | 判断运营复杂度上限 |
| 核心能力标签 | 供应链型 / 内容型 / 运营型 / 品牌型 / 资本型（可多选） | 取证优先级、运营方案侧重 |
| 目标市场 | 国家/平台 | 与卖家原型的匹配度 |
| 风险偏好 | 合规稳健 / 可控擦边 / 高风险短跑 | 已有字段，保持不变 |
| 已有供应链资源 | 自有工厂 / 绑定供应商 / 纯采购 | 供应链可控性评分的信息输入 |
| 已有内容资源 | 自有团队 / 稳定达人池 / 无 | 内容传播维度的信息输入 |
| 基础经营姿态 | 无货源 / 铺货 / 精铺 / 精品 / 品牌 / 规模组合（可混合） | 决定测试漏斗、单品深度和退出机制 |
| 不对称能力 | 供应链 / 产品工程 / 品牌 / 内容 / 广告 / 平台运营 / 履约 / 客户 / 资本 / 数据技术 | 判断哪些约束可能被解除或缓解 |
| 流量与竞争玩法 | 搜索 / 推荐 / 内容 / 达人 / 活动 / 价格 / 合法竞品承接 / 授权跟卖 / 平台允许的多链接实验 | 判断增长机制、合规和窗口寿命 |
| 账号与组织拓扑 | 单店 / 多店 / 品牌矩阵 / 渠道账号 / 国家团队 / 共享中台 | 判断关联、内耗、治理和组织容量 |
| 市场拓扑 | 单国单平台 / 单国多平台 / 多国单平台 / 多国多平台 | 判断协同、迁移折损、固定成本和风险相关性 |

用户未提供卖家画像时，从明确上下文提取候选画像，但不得只凭 SKU 数直接定型。推断不出时保持 **seller-agnostic**：输出产品基础报告，并列 2-3 条条件化路径（例如轻资产验证、精品攻坚、供应链特权），逐条写明成立所需资源和验证问题。不得默认精品型或任何其他玩法，不得直接确定预算比例、首批数量、多平台扩张、重仓动作或低分例外；这些动作必须等待真实资源字段确认。

涉及无货源、铺货/精铺漏斗、供应链独占、品牌路线、竞品流量承接、跟卖、多链接、多店、跨国或多平台协同时，必须读取 [seller-operating-models-and-topologies.md](seller-operating-models-and-topologies.md)。输出基础产品结论与 `Seller-specific Feasibility`，不能把玩法优势直接加到七维总分。

## 决策差异矩阵

同一 CIDM 总分，不同卖家原型的决策解读不同：

| CIDM 总分 | 效率型 | 精品型 | 品牌型 | 规模型 |
|---|---|---|---|---|
| 80-100 建议进入 | 快速上架、抢占活动窗口、价格带卡位；关注周转速度 | 深度 Listing、差异化切入、广告精细运营；关注 Review 积累和利润深度 | 品牌叙事切入、内容矩阵铺设、DTC+平台联动；关注 LTV 和品类延伸 | 组合布局、多平台/多站点并行、系统化运营；关注资金效率和组织承载 |
| 65-79.9 谨慎进入 | 小批量测款、控制单 SKU 投入、快速验证周转 | 小样本测试差异化楔子、验证广告 ROI | 内容测款、品牌故事验证、达人小样本测试 | 分配小预算并行测试、用系统筛选有效候选 |
| 50-64.9 仅观察 | 不建议进入，除非有明确价格优势或活动机会 | 原结论保持仅观察；若满足精品攻坚例外的全部条件，可另开 `Boutique Venture Test`，不得改写为建议进入 | 可观察市场趋势，等待更好的切入窗口 | 纳入观察池，等待品类成熟或竞品失误 |
| <50 不建议进入 | 不进入 | 不进入 | 不进入；但可观察品类趋势作为未来品类规划参考 | 不进入 |

## 评分权重调整建议

七维评分的权重不因卖家类型改变（保持通用评分锚点）。但在**运营方案和投资建议**中，不同卖家类型应侧重不同维度的解读：

| 卖家原型 | 侧重维度 | 原因 |
|---|---|---|
| 效率型 | 供应链可控性 > 利润空间 > 市场需求 | 靠周转和价格竞争，供应链效率和利润是生命线 |
| 精品型 | 竞争可进入性 > 利润空间 > 内容传播 | 靠差异化切入和广告运营，竞争可达性和利润深度决定成败 |
| 品牌型 | 内容传播 > 市场需求 > 机会窗口 | 靠品牌叙事和用户积累，内容传播性和品类空间决定天花板 |
| 规模型 | 利润空间 > 供应链可控性 > 风险可控性 | 靠组合管理和资本效率，利润和供应链稳定性决定规模可持续性 |

"侧重"不改变评分权重，而是：在运营方案中，侧重维度的证据要求更高、深挖更深、运营建议更细。

## 投资姿态映射

投资姿态不是只看总分。严格按以下优先级生成：

1. 门槛红线决定是否停止或降级。
2. 生命周期阶段限定最大动作强度。
3. CIDM 四档决策确定进入、测试、观察或停止。
4. 卖家画像只在上述上限内决定执行方式和资源配置。

即：`最终投资姿态 = 门槛红线 × 生命周期上限 × 分数档位 × 已确认的卖家资源约束`。任何后置规则不得突破前置上限。

### 生命周期动作上限

| 阶段 | 最大动作强度 | 禁止动作 |
|---|---|---|
| LC-1 机会发现 | 无库存研究、VOC、概念与内容信号验证 | 首批备货、重仓、多平台铺开 |
| LC-2 概念验证 | 样品、落地页、创意、供应商打样和小额无库存测试 | 商业批量备货、按高分直接放量 |
| LC-3 小测验证 | 单市场、单平台、小批量测试，按预设门槛迭代 | 未验证单位经济前扩大库存或平台 |
| LC-4 放量期 | 按里程碑逐级释放预算和库存 | 一次性释放全部预算、忽略波动扩仓 |
| LC-5 稳态运营 | 在现金流、利润和风险约束内规模化 | 突破组合风险或供应链承载上限 |
| LC-6 衰退/退出 | 修复、收缩、清仓、转渠道或停止 | 用新增库存掩盖结构性衰退 |

下面的卖家原型映射仅描述执行风格。数量和预算比例是待测算变量，不是跨品类固定处方：

| 决策档位 | 效率型 | 精品型 | 品牌型 | 规模型 |
|---|---|---|---|---|
| 建议进入 | 按周转与失败成本反推单 SKU 上限；先满足生命周期动作上限 | 按差异化验证、盈亏平衡和现金周期反推投入 | 按品牌验证里程碑和 LTV 假设逐级释放资源 | 按组合盈亏平衡、相关性和组织承载分配资源 |
| 谨慎进入 | 单市场低失败成本测试，达到周转门槛后解锁 | 单平台验证楔子与广告 ROI，达到门槛后解锁 | 内容测款优先，品牌假设通过后再备货 | 小预算并行测试，按里程碑解锁 |
| 仅观察 | 不投入 | 不投入，可做 VOC 研究 | 不投入，可做品类趋势研究 | 不投入，纳入观察池 |
| 不建议进入 | 不进入 | 不进入 | 不进入 | 不进入 |

## SKU 与运营策略差异

### 效率型

- SKU 策略：宽 SKU、浅库存、快周转。优先选择可快速上架、活动友好、价格带有竞争力的品类。
- 供应链：公模为主，微调包装或配件。MOQ 敏感，偏好一件代发或小批量。
- 内容：依赖平台自然流量和活动，低成本批量素材。不依赖达人或品牌内容。
- 运营重点：上架效率、价格监控、活动节奏、多平台铺货。
- 退出策略：快速清仓，不恋战。库存深度浅，退出成本低。

### 精品型

- SKU 策略：少量核心 SKU、阶段库存、单品攻坚。不是只挑“天然好品”，还评估可修复缺口能否被产品工程和运营体系转成优势。
- 供应链：多轮打样、私模/工艺/包装定制、CTQ 与批次追溯；用真实销量和预测换阶梯价、账期、产能与物流效率。
- 内容：围绕产品证明构建素材组合和实验矩阵，不依赖单一爆款；内容机制由 VLB 验证。
- 运营重点：调研后形成 G0-G10 主计划，覆盖打样、测品、测款、采购准备、备货、上市、放量、稳态和退出；广告动作由 D09 承接。
- 成本重点：建立规模—成本桥；未来降本保持假设，只有已签合同、账单或成熟批次验证后才进入正式利润。
- 退出策略：先区分产品、供应、页面、内容、广告和履约问题，给有期限的修复窗口；达到 Stop 条件后清仓、转渠道或资产回收，不因沉没成本无限续期。

## 精品型 0→1 攻坚模式

识别到精品型且用户要求完整调研、打爆方案、完整运营计划或调研后直接执行时，必须读取 [boutique-zero-to-one-operating-system.md](boutique-zero-to-one-operating-system.md)。输出产品先天投资质量卡与精品组织攻坚卡，随后按 G0-G10 形成阶段闸门、九条并行工作流、周级作战节奏、资本/库存阶梯和规模—成本桥。

“组织胜率高”不进入七维评分，也不突破生命周期和红线。它只决定是否值得做受控攻坚、资源如何配置、修复窗口多长。运营亏损换规模只有在可承受损失、成本桥、成熟边际路径和退出时间同时明确时才允许；销量本身不是盈利证据。

### 品牌型

- SKU 策略：产品线规划、英雄品+延伸品。优先选择有品类延伸空间、LTV 潜力的品类。
- 供应链：深度定制、品控体系、可能独家供应。MOQ 不敏感，重视品质和一致性。
- 内容：品牌故事、内容矩阵、达人合作、DTC 私域。内容投入高但可复用。
- 运营重点：品牌建设、内容营销、用户运营、全渠道布局。
- 退出策略：评估品牌资产是否可迁移，产品线是否可收缩而非完全退出。

### 规模型

- SKU 策略：组合管理、多品类并行。优先选择组合协同、风险分散、资金效率高的品类。
- 供应链：多品类供应链网络，可能自有工厂。MOQ 不敏感，重视供应稳定性和成本优化。
- 内容：系统化素材生产、专职内容团队。内容投入按 ROI 分配。
- 运营重点：组合收益管理、资金效率、组织承载力、多平台/多站点运营体系。
- 退出策略：评估组合联动影响（退出一个品是否影响其他品的供应链或流量），按组合健康度决策。

## 注意事项

1. **卖家原型是连续光谱，不是硬分类**。一个卖家可能同时具有效率型和精品型特征（如"精铺测试+精品放量"）。允许混合标注，报告中注明主要原型和次要原型。
2. **卖家原型不改变评分，只改变解读**。七维评分模型保持通用，不因卖家类型调整权重或锚点。差异化在决策解读和运营方案层实现。
3. **卖家原型可以进化**。效率型卖家随着经验积累可能转向精品型，精品型可能转向品牌型。报告中可以指出"当前原型下建议 X，若向 Y 型进化则可考虑 Z"。
4. **不要过度推断**。用户未提供卖家画像时保持 seller-agnostic，用条件路径展示不同可行域；不要从品类选择、SKU 数量或一句“运营很强”反推卖家类型。
5. **与组合决策联动**：portfolio-decision.md 的硬淘汰条件"与卖家能力明显不匹配"现在有了具体判断依据——参照卖家原型的预算区间、SKU 容量、供应链深度和核心能力标签。
6. **玩法不是永久身份**：同一卖家可用精铺筛选、精品放量、品牌沉淀和多国扩张组成阶段链；报告必须写当前主玩法、下一阶段玩法及切换 Gate。
7. **高风险玩法不等于优势**：截流、跟卖、多链接和多店先做合法性、授权、商品一致性和平台规则分级；侵权、操纵或规避型方案直接 Blocked。
