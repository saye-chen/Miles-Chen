# 品类投资决策报告

## 目标

默认输出中文《品类投资决策报告》，面向中国跨境电商卖家。不要写泛泛的市场调研。必须通过数据、假设、数学评分和商业推理，判断一个品类是否值得进入。

只有当用户明确要求英文报告时，才输出英文。

默认交付一份专业 Markdown 报告，包含：

- 中文决策摘要
- 结构美观、可复制的中文评分表
- 强制 STEP1-STEP8 中文分析
- 平台进入建议
- 可执行运营打法
- 关键假设与数据缺口

只有当用户明确要求，或用户提供多品类、多平台、多关键词、多竞品原始数据且适合表格建模时，才生成 Excel、CSV 或数据透视表类输出。

生成本地交付文件时，默认保存到本地下载文件夹。除非用户指定其他位置，否则使用 `$HOME/Downloads`。只把最终交付物放进去，例如 Markdown、Excel、CSV 或 PDF 报告。

执行结束前清理临时工作文件、缓存文件、中间导出、截图、抓取原始页和一次性计算文件。不要删除最终交付物或用户提供的源文件。如果无法完成清理，必须说明残留文件和原因。

如果重新生成或优化已有交付物，除非用户明确要求覆盖，否则不要覆盖原文件。新增文件名后缀，例如 `_v2`、`_优化版` 或时间戳。

## 版式规则

默认按中文商业报告审美优化版式，不只追求信息正确。

- Markdown 表格必须清爽，禁止把长推理段落塞进单元格。
- 表格单元格默认不能出现影响美观的多行内容。
- 表格只放短字段：评分、标签、简短判断、短证据。
- 详细推理放在表格下方，用编号说明或短段落呈现。
- 优先使用多个小表，不使用一个很宽的大表。
- 默认控制在 4-6 列。
- 避免大段文字堆叠，使用短段落、留白和清晰分组。
- 关键结论必须使用色块卡片或醒目的 callout。

每个关键结论都要按 1/2/3 优先级标注，并使用不同底色。Markdown 报告中使用 HTML 色块，保证本地文件可读、直观：

```html
<div style="background:#FEE2E2;border-left:6px solid #DC2626;padding:10px 12px;margin:10px 0;">
<strong>1级关键信息｜高优先级/决策红线：</strong> 这里写会直接影响是否进入、是否停止、是否改变投入方式的结论。
</div>
```

```html
<div style="background:#FEF3C7;border-left:6px solid #D97706;padding:10px 12px;margin:10px 0;">
<strong>2级关键信息｜重要判断：</strong> 这里写影响打法、预算、SKU、平台选择或风险控制的结论。
</div>
```

```html
<div style="background:#DBEAFE;border-left:6px solid #2563EB;padding:10px 12px;margin:10px 0;">
<strong>3级关键信息｜辅助观察：</strong> 这里写支持判断但不单独决定进入与否的信息。
</div>
```

默认使用以下中文标签：

- `1级关键信息`: 红底；决策红线、重大风险、最终进入/不进入逻辑。
- `2级关键信息`: 黄底；重要运营打法、预算、SKU、平台选择或风险控制。
- `3级关键信息`: 蓝底；辅助观察、关键假设或背景信息。

硬性要求：最终报告正文必须实际出现多个 1/2/3 级彩色色块，不能只在说明中提到。至少包含：

- 投资决策摘要中 1 个 `1级关键信息`
- 评分总表后 1 个 `2级关键信息`
- STEP1-STEP8 中每一步至少 1 个关键信息色块
- 风险分析中至少 1 个 `1级关键信息`
- 运营打法中至少 1 个 `2级关键信息`

如果输出为本地 Markdown 文件，必须使用上述 HTML `div` 样式实现底色高亮。不能用普通加粗文本替代。

只有用户明确要求英文时，才使用：

- `Level 1 Key Signal`
- `Level 2 Key Signal`
- `Level 3 Key Signal`

## 数据规则

- 需要当前市场、平台、价格、搜索、趋势、竞品、法规或物流数据时，必须联网搜索。
- 优先使用高信号来源：平台页面、Marketplace listing、官方政策、Google Trends、用户提供的卖家工具数据、社媒平台、供应商平台、海关/监管来源、可靠行业数据。
- 必须明确区分“观察数据”和“分析假设”。
- 数据不完整时，可以继续做推理，但必须先标注假设。
- 不要隐藏不确定性。把不确定性转化为风险、敏感性或数据缺口。
- 不要只罗列数据。每个数据点都必须服务于商业推理。

## 前置五道门槛

正式评分前，必须先用“五道门槛”做实战过滤。工具数据主要用于快速筛选第 1、2 道门槛；第 3-5 道门槛必须人工深挖、计算和判断。若门槛结论和加权总分冲突，优先说明门槛红线如何影响最终决策。

| 门槛 | 核心问题 | 默认判断标准 |
|---|---|---|
| 1. 市场够大吗 | 需求体量是否足以支撑规模 | 主关键词与多个相关关键词都有稳定流量；美国 Amazon 普通标品若月搜索量低于 1 万，默认不适合重仓，除非高客单、强复购、专业刚需或平台外流量能补足 |
| 2. 进得去吗 | 新卖家是否有现实曝光机会 | 首页不能全是高评论老品；若有上架不到 12 个月的新品进入首页或拿到可见销量，说明流量仍在流动 |
| 3. 能赚钱吗 | 真实经营成本后是否有净利 | 采购价、头程、履约/FBA、平台佣金、广告、退货损耗全部计入；目标净利润率默认应稳稳超过 20%，否则必须说明降本、提价或降低获客成本路径 |
| 4. 能卖吗 | 合规、认证和侵权是否可控 | 电子类关注 FCC/UL/CE/RoHS 等，玩具关注 CPC/ASTM，食品/接触食品/美妆/医疗相关关注 FDA 或当地监管；专利 FTO、商标和版权风险不能省略 |
| 5. 凭什么是你 | 是否存在可落地差异化 | 必须分析竞品 1-3 星差评、QA 和社媒评论，找出反复出现的痛点；只有能在产品、包装、说明书、套装、材料、尺寸、服务或内容表达上落地改良，才算有效切入点 |

门槛使用规则：

- 不把单一关键词的偶发峰值当成真实需求，必须看关键词簇、趋势和销售信号。
- 不只看评论数量，还要看首页评论结构、上架时间、近期销量、广告位和品牌集中度。
- 利润测算必须做敏感性分析，至少检查广告 ACOS/CPA、退货率、头程/履约费和售价变化对净利的影响。
- 合规和侵权属于决策红线：发现高概率不可控风险时，即使总分达到“谨慎进入”，也可以直接降级为“不建议进入”。
- 没有明确差异化楔子的产品，默认不建议重库存进入，只能观察、轻测或放弃。

6. STEP4：内容传播分析

13. 关键假设与数据缺口

使用专业、有说服力的投委会式中文表达。要简洁、直接、有商业判断。不要使用“有潜力”“竞争激烈”“需要差异化”这类空泛表述，除非同时给出数据依据和商业含义。

## 评分模型

默认使用 100 分加权模型：

| 维度 | 权重 | 判断内容 |
|---|---:|---|
| 市场需求 | 20% | 搜索需求、销量速度、趋势稳定性、购买频次、痛点强度 |
| 竞争强度 | 20% | 竞品数量与质量、评论壁垒、广告压力、Listing 成熟度、品牌集中度 |
| 利润空间 | 20% | 价格带、成本、头程/尾程、履约、平台费、广告费、退货损耗、净利 |
| 内容传播 | 10% | 视觉展示、痛点叙事、达人适配、UGC 潜力、短视频转化逻辑 |
| 供应链可控性 | 10% | 供应商深度、MOQ、交期、质检复杂度、定制空间、包装物流难度 |
| 风险 | 10% | 合规、安全、侵权、平台政策、季节性、退货率、差评风险、监管暴露 |
| 机会窗口 | 10% | 未满足细分需求、竞品弱点、新趋势、平台错配、价格/价值缺口 |

每个维度按 0-10 分打分，再乘以权重。风险维度分数越高，表示风险越低、越适合投资。

默认评分表只使用中文紧凑格式：

| 维度 | 权重 | 得分 | 加权分 | 判断 |
|---|---:|---:|---:|---|

评分表后必须增加“评分依据”，每个维度一条简短解释。不要把长解释塞进表格。

默认决策阈值：

| 总分 | 决策 |
|---:|---|
| 80-100 | 建议进入 |
| 65-79 | 谨慎进入 / 小规模测试 |
| 50-64 | 仅观察或内容测款 |
| <50 | 不建议进入 |

当最终建议和分数阈值不一致时，必须说明原因。例如：某品类总分 72，但如果存在严重合规或侵权风险，仍可建议不进入。

## 分析步骤要求

### STEP1：市场需求分析

判断需求是否真实、稳定，并且具备商业价值。重点分析：

- 搜索量或关键词需求
- 多个相关关键词是否形成关键词簇，而不是单一词偶发峰值
- 销售速度或平台排名信号
- 趋势方向和季节性
- 购买动机和痛点强度
- 复购或配件扩展潜力
- 目标国家的本地使用场景
- 若是美国 Amazon 普通标品，月搜索量低于 1 万时，默认标注为需求规模风险；若仍建议继续分析，必须说明高客单、复购、专业场景、站外内容或平台迁移等补偿逻辑

输出市场需求得分，并说明需求如何影响进入时机。

### STEP2：竞争分析

判断新卖家是否有现实突破机会。重点分析：

- 头部竞品数量、评论深度、评分质量和 Listing 成熟度
- 首页竞品上架时间结构：是否存在上架不到 12 个月的新品进入首页、拿到可见销量或占据广告/自然位
- 品牌集中度和护城河
- 价格带饱和度
- 主关键词难度和广告压力
- 功能、套装、设计、场景、内容或服务上的差异化空间

输出竞争得分。竞争得分越高，代表越容易切入。必须说明哪类新进入者可能赢。若首页几乎全是 2000+ 评论老品且没有新链接突破信号，必须降级进入建议或只建议寻找细分长尾切口。

### STEP3：利润分析

判断该品类在真实经营成本后是否还能赚钱。缺少精确数据时，必须标注假设。

重点分析：

- 售价区间和目标价格带
- 估算成本、包装、头程/尾程、履约、平台费、广告费、退货退款成本
- 毛利率和贡献利润
- 盈亏平衡 ACOS / CPA
- 库存风险和现金周转压力
- 默认完整成本项：采购价、包装、头程、平台履约/FBA 或尾程、平台佣金、广告、退货退款准备金、质检和损耗
- 默认利润红线：目标净利润率应超过 20%；低于 20% 时必须说明可验证的降本、提价、套装化、降低 ACOS/CPA 或降低退货路径，否则不建议进入

可使用简化利润公式：

`估算净利 = 售价 - 产品成本 - 物流/履约 - 平台费 - 广告费 - 退货退款准备金`

输出利润得分，并说明经济模型支持规模化还是只适合小规模测试。必须做敏感性检查：广告成本、退货率、履约费或售价任一关键变量恶化后，净利是否仍能守住合理空间。

### STEP4：内容传播分析

判断产品能否通过内容获得需求，而不是只能依赖搜索广告。重点分析：

- 前后对比效果
- 可展示的痛点
- 短视频开头钩子
- 达人/联盟带货适配度
- UGC 可复制性
- 情绪价值、新奇感、礼品属性或身份表达

输出内容传播得分，并给出 3-5 个内容角度。

### STEP5：供应链分析

判断采购、生产、履约和质量控制是否可控。重点分析：

- 供应商数量和生产成熟度
- MOQ 和交期
- 质检失败点
- 定制或私标空间
- 包装和物流限制
- 售后和瑕疵风险

输出供应链得分，并列出供应链控制点。

### STEP6：风险分析

识别会破坏投资逻辑的风险。重点分析：

- 产品安全、认证和监管规则
- IP、专利、商标、版权和外观侵权
- 平台政策限制
- 高退货/退款原因
- 季节性、流行风险和需求波动
- 差评、投诉和评价风险

输出风险得分。风险得分越高，代表越安全。必须标注可能推翻总分结论的红线风险。合规分析必须具体到品类：例如电子类关注 FCC/UL/CE/RoHS 等，玩具关注 CPC/ASTM，食品/接触食品/美妆/医疗相关关注 FDA 或当地监管；同时必须提醒进行专利 FTO、商标和版权检索。

### STEP7：机会分析

识别新卖家可以赢在哪里。禁止只写“差异化”。必须具体说明：

- 未被满足的细分人群
- 现有竞品 Listing、产品或内容弱点
- 竞品 1-3 星 review、QA、社媒评论中反复出现的痛点，例如松脱、难安装、异味、尺寸不准、说明书看不懂、易损坏、包装差、售后差
- 更好的套装、配件、材料、设计、尺寸或使用场景定位
- 价格缺口或价值缺口
- 平台特定机会
- 时间窗口

输出机会得分，并给出推荐切入楔子。切入楔子必须能落到产品、包装、说明书、套装、材料、尺寸、服务或内容表达中的至少一项；如果只有概念性差异化而无法落地，不能作为进入理由。

### STEP8：最终决策

综合加权得分和商业逻辑，输出明确决策：

- 建议进入
- 谨慎进入 / 小规模测试
- 仅观察或内容测款
- 不建议进入

必须说明推荐的投资姿态，不只评价品类好坏。

## 平台处理规则

如果用户指定平台，报告只围绕该平台输出。

如果用户没有指定平台，从提示词中推断目标国家或地区，并输出该市场相关平台的分结论。美国市场通常比较 Amazon、TikTok Shop、Shopify/DTC、Walmart、eBay、Etsy；其他国家选择当地商业相关平台。

平台表格默认使用中文：

| 平台 | 建议 | 核心原因 | 最佳进入方式 |
|---|---|---|---|

平台结论可以不同于整体品类结论。例如：某品类在 Amazon 因评论壁垒不适合进入，但在 TikTok Shop 因内容钩子强可以小规模测试。

## 运营打法

必须包含可执行运营打法：

- 推荐目标人群
- 推荐价格带
- SKU 和上新策略
- 差异化方向
- 内容角度
- 广告或流量策略
- 供应商和质检要求
- 在假设允许时给出首轮测试预算或测试里程碑
- 明确不建议做的动作

运营打法必须匹配最终决策。例如，如果结论是“谨慎进入”，就推荐小规模测试计划，而不是重库存打法。

## 报告质量标准

每份报告必须做到：

- 给出最终总分，满分 100
- 给出四档决策之一
- 包含加权评分表
- 包含风险判断
- 包含运营建议
- 明确标注假设
- 解释评分背后的推理
- 默认中文输出，除非用户明确要求英文

禁止：

- 跳过 STEP1-STEP8
- 只输出数据不解释
- 没有评分就给结论
- 建议进入但不说明切入楔子
- 使用“提升质量”“做品牌”“差异化”这类空泛建议而不说明具体做法
