# 数据契约与自动化适配

## 标准对象

自动化层必须分开存储自有商品、竞品主体、竞品商品、监控快照、异动事件、机会窗口、原因假设、经营动作和结果回收。所有对象包含稳定 ID、创建/更新时间、来源、证据标签、模型版本和修订记录；新快照追加，不覆盖历史。

## 平台适配器契约

平台适配器只负责把合法取得的数据映射到标准快照，不直接生成原因结论。每次采集返回：平台、市场、商品 ID、采集时间、展示条件、币种、标价/到手价、促销、排名及层级、评分/评论、变体、库存代理、Listing 文本/素材指纹、广告位或内容信号、来源 URL、失败状态。

- 字段不可得时写 `null` 和原因，不用其他平台字段冒充。
- 到手价必须记录券、运费、会员、登录和地区条件。
- 图片、视频和 Listing 变化先生成差异事件，再由分析层判断语义。
- 登录、访问限制、验证码、平台条款或授权不足时停止对应采集，不绕过限制。

## 自动化流水线

`采集/导入 → 模式校验 → 快照追加 → 差异比较 → 阈值候选 → 连续确认 → 归因任务 → 人工复核 → 告警/报告 → 结果回收`。

未经用户明确要求，不创建周期任务、不发送通知、不修改价格/广告/库存。自动检测只证明字段或阈值变化；归因、竞争姿态和高风险动作仍服从证据与人工复核规则。

## 运行质量

记录采集成功率、字段缺失率、重复事件率、告警确认延迟、误报率和基线版本。平台适配器升级后先做回放测试，确认字段含义和历史可比性，再切换模型版本。
