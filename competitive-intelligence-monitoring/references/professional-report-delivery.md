# 专业报告与文件交付

## 目标与边界

正式报告必须能交给老板、业务负责人或投委会直接阅读：30 秒看懂竞争结论，随后能核对证据、判断风险并安排验证。竞争报告提供投资决策的重要输入，但不以竞争评分替代需求、利润、合规、供应链和卖家能力的完整投决。

## 报告层级

| 类型 | 使用场景 | 默认交付 |
|---|---|---|
| Competitive Decision Brief | 快速竞品分析、单品初筛 | 对话简报或 Markdown 文件 |
| Competitive Investment Intelligence Report | 深入、专业、投资级、投委会 | 完整 Markdown 报告 |
| Competitive Monitoring Review | 周报、月报、阶段复盘 | 变化、告警、动作、是否触发重评 |
| Special Intelligence Memo | 价格战、新品、内容、渠道专项 | 单问题证据链与验证方案 |

## 竞争决策页

正式报告必须以决策页开头，至少包含：

| 字段 | 要求 |
|---|---|
| 决策对象 | 自有商品、平台、国家、价格带和时间窗口 |
| 竞争姿态 | 只能使用 `Advance / Test / Watch / Stop` |
| 中文解释 | 推进尽调 / 小规模验证 / 观察等待 / 竞争维度停止 |
| 竞争吸引力 | 结构性机会、可切入性和竞争强度的综合判断，不冒充最终投资分数 |
| 整体置信度 | High / Medium / Low，并指出最弱证据维度 |
| 一句话理由 | 说明为何不是相邻档位 |
| 关键机会 | 最多 3 项，必须能回指证据 |
| 关键红线 | 最多 3 项，写明触发条件 |
| 下一步 | 3-5 个动作，带指标、周期、资源边界和停止条件 |
| 投决状态 | `竞争结论可独立使用` 或 `需路由完整投决` |

### 四档竞争姿态

| 姿态 | 适用条件 | 动作上限 |
|---|---|---|
| Advance | 存在清晰切入楔子，代表竞品缺口有交叉证据，未见竞争性硬阻断 | 推进完整投决或下一阶段验证，不直接重仓 |
| Test | 机会成立但关键差异、获客、价格或证据仍不确定 | 小预算、单变量或最小 SKU 测试 |
| Watch | 当前没有足够优势或窗口，等待价格、评论、上新、渠道等触发信号 | 继续监控，不承诺进入 |
| Stop | 竞争结构已足以否定当前方案，或不存在可验证的差异化路径 | 停止当前方案；若改产品/市场/平台则重新建档 |

低置信度的 `Advance` 自动降级为 `Test`；存在未复核的合规/IP/安全红线时不得输出 `Advance`。

## 完整报告骨架

1. **封面与版本**：报告名称、对象、平台/国家、`CIM-2026.09`、证据截止日、作者角色和保密级别。
2. **竞争决策页**：姿态、置信度、机会、红线、下一步和投决状态。
3. **任务与边界**：用户问题、自有商品识别、卖家约束、时间窗口、未覆盖范围。
4. **方法与证据范围**：查询词、渠道、样本量、去重、排除、采集条件、数据偏差。
5. **竞争格局**：T1-T4、价格带、定位、集中度代理、头部/腰部/新品结构。
6. **代表竞品深拆**：产品、价格、内容、口碑、渠道、运营动作和可复制性。
7. **我方相对竞争力**：CPI/RCG、优势、暴露、差异化可信度；展示权重、缺失值和敏感性。
8. **机会与反例**：人群、场景、任务、承诺-体验落差、替代方案、冲突证据。
9. **动态与情景**：当前基线；有足够快照时再写趋势、变化、原因假设和情景影响。
10. **投资含义**：竞争姿态、最弱假设、哪些结论可用、哪些仍需需求/利润/合规/供应链补证。
11. **验证与行动**：最小测试、指标、周期、预算或资源边界、成功门槛、停止条件和复查日期。
12. **风险与人工复核**：IP、合规、安全、数据偏差、平台展示、恶意竞争指控和大额动作。
13. **Evidence / Assumption Ledger**：核心判断逐条可回溯。
14. **自检摘要**：缺失数据、降级结论、未完成核验、模型版本和文件验证结果。

## 核心表格

| 表格 | 必备列 |
|---|---|
| 自有商品识别表 | 字段、识别值、证据/输入、置信度、决策影响 |
| 竞品分层表 | 对象、层级、CRI、定位、纳入理由、来源、采集时间 |
| 相对竞争表 | 维度、我方、竞品、差距、证据、威胁/机会、动作 |
| 风险表 | 风险、触发条件、影响、置信度、复核角色、进入前动作 |
| 行动表 | 动作、目标指标、周期、资源边界、负责人角色、Go/Stop |

每张核心表后写 1-3 句“所以怎样影响决策”，不得让读者自行猜结论。

## Evidence / Assumption Ledger

Evidence 字段：`E编号`、事实/估算/推断标签、来源标题、平台/机构、URL或文件、数据/访问日期、适用市场、质量等级、支持判断、局限。

Assumption 字段：`A编号`、假设内容、使用位置、当前取值或范围、敏感性、推翻条件、补数动作。

- 每项关键结论至少回指一个 Evidence 或 Assumption 编号。
- 来源冲突时披露冲突、选择理由和对结论的影响。
- 不用链接数量代替证据质量；动态页面注明会变化。

## 文件交付

- 只有用户明确要求下载、保存、报告文件或指定格式时生成本地文件；未要求文件时在对话中交付。
- 用户只说“可下载专业报告”但未指定格式时，默认生成一个完整 Markdown 文件，不额外生成其他格式。
- 未指定保存路径时保存到 `$HOME/Downloads`，文件名使用 `CIM-<对象>-<平台>-<国家>-<YYYYMMDD>.md`；同名时增加时间戳，不覆盖旧文件。
- DOCX、PDF、XLSX 等其他格式不在本 Skill 内扩充；用户明确指定其他格式时，统一服从当前环境的全局文件交付约束和对应专业能力。
- 报告必须包含决策页、正文、Evidence/Assumption Ledger 和自检摘要；不得把聊天内容原样复制成文件。
- Markdown 交付前检查标题层级、目录锚点、表格列数、代码块闭合、本地与外部链接、图片路径、币种/日期/小数口径、Evidence/Assumption 编号回指和敏感信息。
- 最终回复提供 Markdown 文件的可点击绝对路径，并概括结论、证据截止日和主要限制。

## 专业表达

- 区分观察数据、用户输入、计算结果、分析假设、推断和决策含义。
- 避免“竞争激烈、很有潜力、建议优化”等空话；紧跟证据、对象、动作、指标和期限。
- 失败、观察和停止都是有效专业结论，不为了积极而软化。
- 无利润数据时不得回答“钱能不能赚”；应说明竞争上是否值得推进以及完成最终投决还缺什么。
