# 归因学习与结果回收

## 归因验证记录

每条原因假设保留 `hypothesis_id`、事件、原证据、反驳信号、原置信度、预计验证时间、验证结果、修订原因和模型版本。到期后只能标记 `validated / falsified / inconclusive`，不静默覆盖原判断。

## 经营结果回收

按 `竞品异动 → 原因假设 → 我方动作 → 我方结果 → 替代解释 → 结论回写` 记录。结果改善不自动证明我方动作有效；同期促销、季节、平台流量、库存、价格和其他运营动作都要作为替代解释检查。

## 质量指标

- 异动有效率：确认真实的告警数 / 已复核告警数。
- 高置信归因命中率：最终验证的高置信归因数 / 已到验证期的高置信归因数。
- 告警误报率：确认数据或展示噪音的告警数 / 已复核告警数。
- 建议动作验证有效率：达到预设成功信号的动作数 / 已完成验证动作数。

样本不足时报告分子、分母和事件明细，不输出稳定百分比。指标用于改进规则和阈值，不用于掩盖未复核事件。
