# 内容创意与传播决策协议

## 能力章程与边界

本 Skill 的对象扩展为短视频、直播切片、静态图、轮播、UGC、测评、教程、品牌片及其素材组合。拥有内容观察、内容机制、创意策略、生产规格、测试变体、疲劳与商业迁移判断主权；不拥有达人商务、广告预算、品牌总定位或品类投资主权。

无法直接读取完整素材时降低观察粒度：只分析可见帧、文案或页面，不推断完整对白、节奏、音效、留存或成交。

## 决策链

1. 固定对象、格式、平台、国家、品类、受众、生命周期和传播目标。
2. 区分观察事实、元数据、用户输入、基准、推断与不可知项。
3. 映射用户决策阶段：注意 → 理解 → 信任 → 比较 → 行动 → 使用/复购。
4. 识别创意母题、产品证明、风险逆转、社会证明、情绪/身份表达、Offer 与 CTA。
5. 检查产品事实、宣称、版权/肖像、平台与国家合规红线。
6. 输出可生产 Brief、脚本/画面结构、难度、成本区间和验收标准。
7. 设计正交优先的创意实验与停止条件。
8. 配置探索、主力、防疲劳和再营销素材，并定义刷新触发器。
9. 评估自然→广告、达人→品牌资产、跨平台/跨国家迁移等级。

## 创意策略地图

每个策略项包含：目标决策阶段、用户阻力、核心命题、所需产品证据、表达机制、适用格式、风险、反例和验证方式。创意分数只描述内容质量或机制适配，不证明销量、增量或竞争优势。

## 素材生产合同

Brief 至少包含：单一目标、受众/场景、必须展示的产品事实、禁止宣称、Hook、证明、Payoff、CTA、格式/时长/比例、语言与文化校准、拍摄资产、制作难度、成本/交期、交付文件、验收门槛和可变元素。

不得用“爆款感”“高级感”等不可验收词替代镜头、声音、文案、产品证明和交付标准。

## 创意测试矩阵

候选变量：Hook、卖点、人物、场景、证明方式、Offer、CTA、节奏和格式。先写最弱假设与主要指标，再选择设计，不以“多做几个版本”代替实验。

### 实验设计选择

| 条件 | 默认设计 | 允许同时变化 | 可下结论 |
|---|---|---:|---|
| 预算小、只验证一个最弱假设 | OFAT / A-B | 1 个主变量 | 该变量在当前对象和窗口的描述性或因果结果 |
| 两个变量可能有交互且四个组合可交付 | `2×2` 全因子 | 2 个变量 | 主效应与交互；每个组合都必须有足够信息量 |
| 变量多、预算受限 | 预先指定的部分因子或分轮筛选 | 每轮不超过设计矩阵规定 | 仅估计预先声明的可识别效应，不把混杂效应拆开解释 |
| 平台分发不可控、未随机或样本不均衡 | 描述性 matched-cell | 可观察多个，但不得作因果拆分 | 候选排序与下一轮实验，不声称增量 |

实验前冻结：实验单位（用户、素材、广告组或发布时段）、分流机制、基线、主要指标、机制指标、护栏、最小实际意义 `MDE`、最大预算、最大窗口、排除规则和停止规则。素材是处理，不等于实验单位；同一用户反复看到多个版本时必须披露串扰，必要时按用户或地域聚类。

### 样本与停止

- 不使用跨平台固定“最小曝光量”。样本需求由基线率、MDE、方差、显著性或后验可信度、检验功效、实验单位和聚类效应共同确定。
- 无法可靠估计样本时，预先设定最大信息预算和最小方向性门槛，结论降级为探索性；不能把“没有显著差异”写成等效。
- 只允许按预先声明的序贯规则提前停止；不得每天查看结果后在有利时截断。
- 多素材、多指标同时比较时，预先指定唯一主要指标，并采用错误发现率、族错误率或分层贝叶斯收缩之一；未经校正的赢家只进入复验。
- 平台未按计划分发、样本比例严重失衡、追踪断裂、库存/价格/落地页同期改变时，实验标记 `invalid` 或 `inconclusive`，不得评出赢家。

### 决策与复验

结果必须区分 `winner / iterate / no-difference / harmful / inconclusive`。胜出要求主要指标达到最小实际意义、护栏未破、产品证明未削弱且结果在预设窗口成熟；首次胜出只进入复验或受控主力池。没有可靠实验或准实验时，只报告描述性表现，不声称增量。

## 素材组合与疲劳

- 探索素材：覆盖不同机制，购买信息而非追求短期稳定。
- 主力素材：已通过门槛，承担规模交付。
- 防疲劳素材：保留核心证明，改变开头、人物、场景或节奏。
- 再营销素材：回答已知阻力，不重复冷启动教育。

区分受众饱和、创意磨损、Offer 失效、平台分发变化和产品/页面承接问题。仅凭频次或 CTR 下降不得自动判定创意疲劳；需对比受众、投放、页面、价格和库存变化。

素材状态固定为 `candidate → exploring → validated → core → fatigue-watch → refresh/retire`。每次迁移记录证据窗口、主要指标、护栏、成本、授权期限和负责人；不得因一次短窗胜出直接进入 `core`。

疲劳判定至少同时满足：同一可比受众/版位内连续窗口恶化、曝光或频次达到可解释水平、Offer/价格/库存/页面没有足以解释变化的同期改变。刷新时保留一个核心证明锚，只改变预先声明的 Hook、人物、场景或节奏；若核心证明也改变，则按新候选重新建立基线。

## 商业迁移等级

| 等级 | 迁移 | 默认要求 |
|---|---|---|
| L1 | 同平台同国家同品类轻改 | 验证新 Hook/人物不破坏证明链 |
| L2 | 自然内容转广告或达人转品牌资产 | 补授权、披露、CTA、落地页与付费环境测试 |
| L3 | 跨平台 | 重做格式、节奏、交互和承接链 |
| L4 | 跨国家/语言 | 重做文化、语言、演员、价格、宣称与合规校准 |
| L5 | 跨品类 | 只迁移抽象机制，产品证明与用户阻力全部重验 |

每次迁移必须输出源对象、目标对象、保持项、重验项、不可迁移项、授权/合规差异、失败回滚和证据状态。L2-L5 不得继承源素材的增量幅度；L4-L5 默认从 `candidate` 重新开始。

## 标准输出

输出内容策略地图、素材组合、生产 Brief、实验矩阵或迁移卡之一；无论层级均保留对象、观察证据、机制、反例、商业/合规约束、动作、成功与停止条件。跨域交付使用 `proposed / validated / blocked / inconclusive`，明确允许和禁止用途。
