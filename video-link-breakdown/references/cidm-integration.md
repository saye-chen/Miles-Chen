# CIDM 对接

只有用户提供品类投资报告或明确要求回写时启用。VLB 独立运行时不读取或修改 CIDM 历史报告。

两个 Skill 使用统一的交付层级命名：Decision Card / Decision Memo / Investment Diligence。回写报告的交付层级应与原 CIDM 报告保持一致或升级，不降级。

## 输入

读取 CIDM 模型版本、证据截止日、内容传播评分、利润评分、风险评分、切入楔子、最弱假设和五道门槛状态。

### 产品一致性前置检查

在开始任何分析之前，先比对以下三个产品是否一致（比对维度：品类、细分款式、价格带、核心功能机制）：

| 产品 | 来源 | 示例 |
|---|---|---|
| **A. CIDM 目标产品** | CIDM 报告中评估的产品 | 宠物智能饮水机（室内电子款，$25-60） |
| **B. VLB 视频产品** | 被拆解视频中展示的产品 | 自动补水宠物饮水碗（户外机械款，$15-25） |
| **C. VLB 复刻目标产品** | 用户要求评估复刻可行性的产品 | 便携搅拌杯（$25） |

根据一致性结果分流：

| 情况 | 处理 |
|---|---|
| **A = B = C**（三者一致） | 正常流程。迁移等级自动判定为"同品"，回写置信度不受损 |
| **B ≠ A，但 C = A**（视频是不同产品，复刻目标是 CIDM 产品） | 提醒用户："视频展示的产品与投资决策的产品不同，拆解结论的迁移性有限，建议更换为同产品视频。"用户确认后继续，迁移等级按"产品可迁移性评估"判定 |
| **A、B、C 三者互不相同** | 强烈建议用户重新选择视频或复刻目标。如用户坚持继续，回写报告只输出"打法参考"，不做维度调整 |

> 此检查的目的是在分析开始前拦截产品错位，避免用户做完一整套分析后才发现结论无法回写。提醒不等于阻断——用户确认后可以继续，但必须知道分析价值受限。

## 产品可迁移性评估（回写阶段）

在做任何维度调整之前，必须先判断 VLB 的视频证据对 CIDM 目标产品的可迁移性。这是回写报告的逻辑起点——如果迁移性不成立，后续所有调整都失去基础。

如果前置检查已判定为"同品"，此步骤自动通过，无需额外评估。

| 迁移等级 | 条件 | 对调整的影响 |
|---|---|---|
| **同品** | VLB 分析的视频与 CIDM 目标产品属于同一细分 SKU（同品类、同功能机制、同价格带） | 维度调整正常生效，置信度不受损 |
| **相近品** | 同品类但不同细分（如机械款 vs 电子款、户外 vs 室内、$15 vs $45） | 调整幅度上限 ±1 分；所有调整置信度降一级（确定→中等，中等→低）；必须给出迁移性验证动作 |
| **跨品类** | 不同品类（如宠物用品 vs 厨房小家电） | 不做维度调整；只输出"打法参考"——提炼可迁移的内容策略模式，但明确标注不构成评分依据 |

评估输出格式：

```
迁移等级：[同品 / 相近品 / 跨品类]
产品差异：[具体差异点，如"机械浮球阀 vs WiFi 电子水泵"、"户外无电源 vs 室内插电"]
视觉机制可迁移性：[高/中/低] — [理由]
```

当迁移等级为"相近品"或"跨品类"时，回写报告必须在调整页之前展示此评估，并在每个调整项中注明迁移性对置信度的影响。

## 回写映射

- 产品-视频匹配与可复制性 → 内容传播；
- 单位经济和达人/素材成本 → 利润空间；
- 产品与内容合规 → 风险可控性；
- 新视频证据 → 切入楔子、验证计划或机会窗口。

## 输出协议

输出结构化”建议调整”，按以下顺序组织：

1. **产品可迁移性评估**（前置）：迁移等级、产品差异、视觉机制可迁移性判断。此节为后续所有调整的逻辑基础——如果迁移性不成立，调整不应生效。
2. **回写总览**：维度调整汇总表，标注每项调整是否受迁移性影响。
3. **逐项调整说明**：原 CIDM 版本、原分、建议分、新增证据/假设编号、调整理由、置信度（受迁移等级约束）、迁移性对该项的具体影响。
4. **五道门槛重判**：是否触发门槛状态变化。
5. **迁移性验证动作**：当迁移等级为”相近品”时，必须给出验证迁移性的具体实验（如拍摄目标产品视频测试效果），含 Go/Stop 条件。验证失败时的回退规则。
6. **Evidence / Assumption Ledger**：新增证据和假设，标注来源和局限性。

不静默覆盖历史报告，不因 CIDM 原结论乐观而弱化 VLB 发现的内容、利润或合规否决项。未经用户确认不得修改旧评分或文件。
